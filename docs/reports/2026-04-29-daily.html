今日最大的事：Ghostty 宣布离开 GitHub

来源： Hacker News · 1817 points / 573 comments · 2026-04-28 原文： https://mitchellh.com/writing/ghostty-leaving-github

这件事是什么

Ghostty 创始人 Mitchell Hashimoto（GitHub 第 1299 号用户，2008 年注册）宣布，经过 18 年的重度使用，他决定将自己创立的终端模拟器 Ghostty 从 GitHub 迁移走。Hashimoto 在博文中表达了强烈的情感：GitHub 曾是他"比一个人应该爱一件东西还要爱"的地方——他在失恋、蜜月清晨、大学深夜都在 GitHub 上写代码。但过去一个月他记录了每天的 GitHub 故障，"几乎每天都有 X"。GitHub Actions 的一次故障让他 2 小时无法做 PR Review，这让他意识到"这不再是严肃工作的地方"。

为什么这么热

这不是一个简单的平台迁移决定，而是一个标志性信号： 1. GitHub 可靠性危机——第三方监测显示 GitHub 过去 90 天可用率仅 90.21%，远低于承诺的 99.9%。这不是个案，是系统性问题 2. Ghostty 地位特殊——这是 HashiCorp 联合创始人 Mitchell Hashimoto 创立的明星项目，他同时创造了 Vagrant、Terraform 等工具 3. 时机敏感——就在 4 月 27 日 GitHub 经历了一次大规模 Elasticsearch 故障后（虽然 Hashimoto 强调决定早就做出）

各方声音

支持/兴奋： - "GitHub 的稳定性确实越来越糟，Ghostty 不是第一个离开的" —— byteiota 报道指出 Zig 语言已在 2025 年 12 月迁移到 Codeberg - "技术迁移的障碍从未如此低，替代平台已成熟" —— DEV Community 开发者

质疑/反对： - "网络效应依然存在——1 亿开发者留在 GitHub 是因为其他人都在" —— 评论指出迁移成本被低估 - "Hashimoto 是在用离开来倒逼 GitHub 改进" —— 有人认为这是"以退为进"

背景上下文

这不是孤立事件。2025 年 12 月，Zig 编程语言因 GitHub "腐烂的工程文化和 AI 痴迷"迁移到 Codeberg。Ghostty 之前刚宣布成为非营利组织（由 Hack Club 财务支持），现在再加上离开 GitHub，清晰表明 Hashimoto 正在构建一个不依赖任何单一商业平台的可持续开源项目。GitHub 被微软收购后，越来越多开发者质疑其优先级的转变——AI 功能（如 Copilot）被大力推动，而基础设施稳定性被忽视。

【判断】

对独立创业者/AI产品人有两个信号： 1. 基础设施去中心化正在加速——GitHub 的垄断地位正在被挑战，Codeberg、GitLab、Forgejo 等替代方案正在成熟。对于构建 AI 工具的团队，这意味着需要考虑多平台策略，不要把所有资产绑在一个平台上。 2. 可靠性即产品——Ghostty 事件提醒我们，即使是最大的平台也可能让你失望。如果你的产品依赖第三方 API 或服务，需要有多供应商策略。Hashimoto 的日志方法（每天记录故障）也值得借鉴：用数据驱动决策，而非仅凭感觉。

ProductHunt 2026-04-28 榜单

Clera｜406v

AI 驱动的候选人代理，帮你找到合适的初创职位

是什么 Clera 是一个"AI 人才代理"，面向求职者的招聘平台。不同于传统招聘网站（雇主付费），Clera 代表候选人——你的职业目标优先。AI 分析你的技能和经验，匹配 a16z、YC、Sequoia 等顶级投资机构支持的初创公司。

解决什么问题 传统招聘流程中，候选人需要海投简历、经历漫长的等待和"幽灵"（无回复）。Clera 承诺"2 分钟而非 2 小时"完成设置，直接通过 Slack 向创始人/招聘经理介绍你。

用户反馈 Trustpilot 上评分 4.3/5，41 条评价。好评："匹配引擎很棒，加速了申请流程"。差评：有用户投诉收到"冒充 VIP"的邮件，质疑其营销方式是否透明。

【判断】 Clera 的核心赌注是"候选人优先"模式。招聘行业的信息不对称一直存在——雇主知道他们要什么，但候选人不知道哪些公司真正适合自己。Clera 的 AI 试图弥合这个鸿沟。最大风险是信任问题——如果用户觉得被"虚假个人化"邮件骚扰，口碑会迅速恶化。

SureThing.io｜327v

具有长期记忆的自主 Agent，像人类一样沟通结果

是什么 SureThing 是一个 24/7 运行的自主 AI Agent，核心差异化是"状态级 Agentic 记忆系统"——它记住你的偏好、沟通风格、项目目标，跨会话保持一致性。可以自动处理邮件、日历、项目管理，无需持续提示。

解决什么问题 传统 AI 助手（ChatGPT、Claude）每次对话都"失忆"，用户需要反复解释。SureThing 声称"说一次，永远记住"， onboarding 后记忆深度会持续改进。

怎么做到的 - 基准测试：记忆深度 SureThing 84.8 vs ChatGPT 40 vs OpenClaw 60 - 集成 1000+ 应用 - SOC 2 认证和 Google CASA Tier 2 合规 - "人机回环"设计——Agent 自主行动，但关键决策需要用户确认

用户反馈 声称用户每周节省 15+ 小时，自主处理 40% 的常规邮件和日程请求。一位技术总监评价："AI 代理的 Mac"。邀请制——你需要与 AI 对话才能获得邀请码， onboarding 本身就是产品演示。

【判断】 SureThing 代表了 AI Agent 的下一个阶段：从"反应式聊天"到"主动式代理"。其核心赌注是记忆层的工程化——如果真能实现"长期记忆"而不产生幻觉，这是一个真正的护城河。但风险在于夸大宣传——记忆深度很难量化验证，且 SOC 2 认证不等于用户信任。

Social Fetch｜229v

通过 API 从任何社交平台获取实时数据

是什么 Social Fetch 是一个社交媒体数据 API，覆盖 TikTok、Instagram、YouTube、X、LinkedIn、Facebook 等 20+ 平台。统一端点设计，相同的 JSON 响应格式，相同的认证方式（x-api-key）。

解决什么问题 开发者和企业需要社交媒体数据来构建分析工具、监控品牌声誉、发现影响者。自建爬虫维护成本高且不稳定。Social Fetch 提供"零维护"的数据管道。

怎么做到的 - 按调用计费，积分永不过期 - 无订阅费用，即用即付 - 提供 TypeScript SDK 和完整类型支持 - 实时数据（非缓存）

【判断】 这是一个基础设施产品，价值在于减少开发者维护爬虫的时间。风险在于平台政策变化——Twitter/X、LinkedIn 等平台的 API 政策经常收紧，依赖第三方数据有合规风险。适合需要快速原型验证的团队，长期大规模使用可能需要自建方案。
