## GLM-5.1：为什么 HN 420分？讲清楚

**这个模型具体强在哪**

GLM-5.1 是智谱 AI 发布的开源模型（MoE 架构），宣称专为 Long-Horizon Tasks 优化——即需要多步骤、跨会话、长时间执行的复杂任务，比如"自主写一个完整项目"或"连续 agent 执行数小时任务"。

HN 评论区的真实测试结果：

**做到了的：**
- @stavros：「模型独立找到了一个 SQL 注入漏洞来解决网球预定系统的问题，我没有提示它用这个方法」
- @InsideOutSanta：「Opus 排不上的时候我用它作为 fallback，开源模型里质量最好的之一」
- @DeathArrow：「强烈建议取消 Cursor 订阅，用 z.ai Pro 跑 GLM-5.1，性能差不多但便宜很多」
- @disiplus：「它解决了 Kimi K2.5 解决不了的任务」

**没做到的：**
- @bigyabai：「上下文超过 128k token 之后模型开始失去连贯性」
- @KaoruAoiShiho：「context window 退化是个真实问题，长任务恰恰最需要长上下文」
- @XCSme：「通用智力 benchmark 上，GLM-5.1 比 GLM-5 还差」
- @ummzokbro：「z.ai 基础设施改动之后变得不稳定，有文件损坏风险」
- @embedding-shape：「context 退化问题直接推翻了它'擅长长任务'的核心主张」

**结论：** HN 高分的原因是"真实可用"——有人用它替代 Opus 和 Cursor 在跑真实任务，而不只是 benchmark。但它的最大问题恰好是它声称的强项：超过 128k token 后表现下降，而长任务恰恰需要长上下文。保持 100k 以内+context 压缩的情况下，和 Opus 差不多；超出后不稳定。

## ProductHunt 4月7日（周一）深度榜单

### NovaVoice｜475v｜语音操作系统

**是什么：** macOS/Windows 的全局语音 OS——不是普通的语音转文字，是"说话控制电脑"的完整系统。

**解决什么问题：** 三个场景：① 打字慢（说话 200+ wpm vs 打字 60wpm）② 在当前 app 里完成跨 app 操作，不用切换 ③ 语音提问不用开浏览器

**怎么做的：** 全局热键激活 → 截图当前屏幕上下文 → 语音转文字 + AI 理解意图 → 执行操作（dictation / app 控制 / AI 回答）。格式会根据当前 app 自动适配（在 Gmail 里输出邮件格式，在 Slack 里输出对话格式）。

**用户反馈：** 产品4月7日当天发布，PH 暂无用户评论（太新）。对标产品可参考：同类产品 Wispr Flow 的用户反馈集中在"dictation 精度高但 app 控制不稳定"，NovaVoice 强调了 app 控制和 AI 整合，值得观察是否解决了这个问题。

---

### OpenOwl｜229v｜本地运行的 AI 桌面自动化

**是什么：** 让 Claude / Codex / 任何 MCP 兼容 AI 直接操作你的屏幕——看界面、点按钮、填表单、跨 app 操作。

**解决什么问题：** API 做不到的事——没有 API 的遗留系统、需要登录的网页操作、任何要"点来点去"的重复任务。典型场景：LinkedIn 挖潜客、Shopify 后台批量更新、老 CRM 数据录入。

**怎么做的：** 用自然语言描述任务 → OpenOwl 接管屏幕 → 调用 AI 理解界面 → 执行操作。完全本地运行，不走云端。

**用户反馈：** 发布当天 229v 但暂无文字评论，说明用户在"等着看"而非有过深度使用体验。本地运行 + MCP 兼容是差异点，但 Computer Use 类产品的核心挑战一直是稳定性，这个要等真实用户用几周才能判断。

### Google AI Edge Eloquent｜164v｜离线 AI 语音听写

**是什么：** Google 出品，基于 Gemma 的离线语音听写工具，标榜"完全不联网"。

**解决什么问题：** 对隐私敏感用户（医生、律师、记者）的语音输入需求——不想把声音发给 OpenAI/Google 服务器，想在本机跑。

**怎么做的：** Gemma 模型在设备端跑推理，不走任何云 API，数据不离开本机。

**用户反馈：** PH 评论区反应：
- 多位用户的第一反应是「终于有人做这个了」，说明隐私需求是真实存在的
- 有人指出「Google 出这个很讽刺——你信任 Google 的隐私承诺吗？」（评论区最高赞之一）
- 技术用户关注 Gemma 在离线场景的实际精度，与 Whisper 的对比还没有结论

**【判断】** 讽刺感是真实的——Google 的"隐私"定位需要用户相信他们不会通过 OS 层收集数据。但如果精度过关，这对 Apple 的 Siri 是直接压力：Google 在用开源模型做 Apple 引以为傲的"端侧 AI"。

### Caret｜162v｜Mac 任意输入框按 Tab 唤起 AI

**是什么：** 在 Mac 任何 app 的任何输入框里，按 Tab 键唤起 AI 补全/修改。

**解决什么问题：** 想在 Notion、邮件、Slack、任何 app 里用 AI 写东西，但不想切到 Claude.ai 或 ChatGPT 页面来回复制。

**怎么做的：** 全局键盘 hook + 截图上下文 + AI 补全，结果直接输出到光标位置。

**用户反馈：** 发布当天 162v，21条评论，主要声音：
- 「终于有人做了 Raycast AI 一直没做好的那件事」
- 「和 Ghostwriter / Superwhisper 有什么区别？」（问差异化的占多数）
- 制作者回复：「区别是我们读取上下文，不只是光标位置」

**【判断】** 这是今年第5-6个"Mac 全局 AI 快捷键"产品了。需求是真实的，但这个赛道的天花板是 Apple 自己——如果 macOS 下一版原生支持，这类产品会瞬间失去意义。在 WWDC 前做这个赛道有时间窗口风险。
