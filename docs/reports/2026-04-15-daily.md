HN 580分 + 573分（两条合计 1153 分），4/14 当日最热话题。

## Claude Code Routines：Anthropic 让 AI 自己跑 cron job

HN 565分 / 334评论。

Anthropic 发布 Claude Code Routines（研究预览版）——本质是**可自动执行的 Claude Code 配置**：一个 prompt + 一个仓库 + 一组 connectors，打包后可以：
- 按固定时间表运行
- 通过 API 触发
- 响应 GitHub 事件（webhook 驱动）

典型场景：CI/CD 输出扫描→错误检查→自动发报告；alert 分诊→自动归类并建 ticket。运行在 Anthropic 云端，不需要你的 Mac 在线。

配额限制：Pro 用户 5 次/天，Max 15 次，Team/Enterprise 25 次。

同时发布了 Claude Code 桌面端重设计：集成终端、更快的 diff viewer、文件编辑器、扩展预览区。

**【判断】** Routines 的本质是把 Claude Code 从"人在终端前交互"扩展到"无人值守自动化"——从 co-pilot 到 auto-pilot 的关键一步。334 条评论的热度说明开发者对此非常兴奋。但 Pro 用户每天只有 5 次配额，叠加上周的配额耗尽争议，Anthropic 的"给你强大工具但限制使用量"策略正在制造越来越大的摩擦。

### Figma for Agents｜490v（日冠）

Figma 正式开放画布给 AI agent——agent 现在可以直接在 Figma 中读取和操作真实的设计文件、组件和 design token。通过 MCP server 集成，任何支持 MCP 的 AI 工具（ChatGPT、Claude Desktop、Cursor）都能写入 Figma 文件。

**【判断】** 这是设计工具领域的"GitHub Copilot 时刻"——从"AI 辅助设计"进化到"AI 直接操作设计文件"。490v 是今天的绝对王者。Skills 系统的设计思路值得关注：**给 agent 设定边界和规范，而不是无限制自由生成**——这个模式适用于所有 agent 产品。

### Recall 2.0｜PH 208v｜核心赛道 · 直接竞品

Recall 2.0 今天在 PH 上线。核心功能：
- **Chat with Your Knowledge**：跨所有保存内容的 AI 问答，可以选择"优先用我的知识"还是"引入互联网补充"
- **Quiz 2.0**：开放式题目、闪卡、定时测验——把知识管理游戏化
- **Graph View 2.0**：知识图谱可视化，看到概念之间的连接，发现新路径

50 万用户，180 个国家。Chrome 扩展可用。

**和我们的关系：** 这是目前最接近我们产品愿景的竞品之一——"保存内容 → AI 理解 → 知识可查询"。**关键差异：**
1. **信号源不同**：Recall 靠用户主动保存内容，我们靠用户在阅读中的自然批注（划线 = 意图信号，不只是收藏）
2. **理解深度不同**：Recall 的 AI 是检索式的（"你保存了什么我就搜什么"），我们的目标是理解式的（"你为什么对这段话感兴趣"）
3. **Graph View 是我们可以借鉴的**：知识图谱可视化对用户来说是很有感召力的功能——让你"看到"自己的知识结构
4. **208v + 50 万用户 = 需求已验证**，但他们的 Chrome 扩展评分和评论值得去看一下，判断真实使用深度

### Figma for Agents｜PH 490v｜可借鉴模式

Figma 的 Skills 系统——让 agent 在特定规范内工作——和我们的四层 Context 架构（L1 文化层 + L2 项目层）是同一个思路：**不是给 agent 无限自由，而是给 agent 正确的约束**。

**和我们的关系：** 不是竞品，但设计思路高度相关。Figma 用 Skills 确保 agent 产出符合品牌标准；我们用 L1/L2 Context 确保 agent 回复符合用户思考方式。核心原则一样：**agent 需要被"教"如何在特定上下文中工作**。490v 验证了这个模式。
