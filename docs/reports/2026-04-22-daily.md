## 今日最大的事：Anthropic 悄悄把 Claude Code 从 Pro 摘掉，当晚就撤回

**是什么：** 4 月 21 日 Anthropic 官网 pricing 页面和 support 文档静默更新，Claude Code 从 $20/月 Pro 档被移除，只留在 $100/月 Max 5x 和 $200/月 Max 20x 档。没有 blog、没有邮件、没有 changelog。当天晚些时候页面又被悄悄改回。Growth 负责人 Amol Avasare 事后在 X 解释这是"对 ~2% 新 prosumer 注册用户的 A/B 测试"。HN 主帖 [546p / 263c](https://news.ycombinator.com/item?id=47684091) + 续帖 [263p / 104c](https://news.ycombinator.com/item?id=47685247) 双双霸榜。

**为什么热：**
- Simon Willison 当天连发两篇博客，明确表示"对 Anthropic 价格透明度的信任已经动摇"，不敢再推荐依赖 $100/月订阅的教程课程
- 本周第二次 Anthropic 被批（上周 Opus 4.7 token 涨 46% 无公告已经是第一次），"用户信任被两次消耗"成为主旋律
- 操作手法本身——用真实用户做定价 A/B 测试但不告知——被广泛定性为"背信"而不是"调整"

**各方声音：**
- 反对方：Ed Zitron（Where's Your Ed At）痛批"这不是定价变化，是欺骗"；Reddit r/ClaudeAI 大量用户表示"正在迁到 Copilot / Cline / 本地模型"
- 中立：Simon Willison 认为 $100/月 实际大概率不会发生，"但沟通混乱本身就是问题"
- 支持方：基本没有。Anthropic 自己的解释也只是"测试，不影响现有订阅"

**背景：** Anthropic 从 2025 年下半年开始反复收紧：Max 套餐上线、Claude Code usage cap 调整、第三方框架（如 OpenClaw）被禁止走订阅额度。同日 GitHub 宣布 Copilot Individual 计划大改（暂停新注册、Opus 4.7 限 $39 Pro+ 独占、转 token-based billing）——**"seat-based pricing 在多 agent 工作流下崩塌"在 Anthropic 和 Microsoft 同一天被公开承认**。

**【判断】：** 对独立创业者是三重信号：(1) 订阅型 AI 工具的定价稳定性不能作为长期假设，做产品规划时要预留供应商换血预算；(2) "悄悄改定价"的手法会被 HN/Simon Willison 级别的社区立即曝光——mem-ai 将来做订阅定价时，任何调整都必须走明牌 + grandfather 老用户；(3) 用户对"被当成实验对象"极度敏感，A/B 测试定价是红线。**对 mem-ai 产品设计的直接影响**：必须从 Day 1 设计 BYO API key 路径，等厂商完成定价转型再适配是被动。

## 次要热点 2：Meta 强制记录员工键盘鼠标训练 AI，内部称其"反乌托邦"｜HN 529p

**是什么：** 路透独家爆料，Meta 向美国员工发内部备忘录，宣布部署名为 **Model Capability Initiative (MCI)** 的工具，在工作设备上记录键盘输入、鼠标移动、点击、以及偶尔的屏幕截图，用于训练 AI agent。发言人 Andy Stone 表示数据仅用于模型训练、有"敏感内容保护措施"，但未说明具体排除范围，**且没有 opt-out 选项**。注意：这是追踪**自己员工**，不是普通用户。

**为什么热：**
- 时间点敏感：Meta 今年已经多轮裁员，同时员工被要求"贡献自己的工作流"训练那个将要替代他们工作的 AI agent——叙事天然有戏剧张力
- 研究者 Ifeoma Ajunwa 点破：这把白领工作者降级到"外卖骑手、仓库工人"那种实时监控的境地
- Meta 最需要留住的资深工程师/研究员恰恰是最可能对这种追踪说不的人——自损人才战

**各方声音：**
- 反对方：匿名 Meta 员工对路透说"very dystopian"；Slashdot/HN 评论区讨论"如果 AI agent 目标是替代人的点击，那么'标注数据'的人其实是在标注自己的解雇书"
- 中立：Stone 强调"不用于绩效考核"，但员工普遍不相信 scope 不会扩散
- 支持方：技术上合理——要让 agent 学会点 dropdown、用快捷键，就是得有真实操作数据。但几乎所有技术媒体都指出"合理"和"可接受"是两回事

**【判断】：** 对 mem-ai 两层启示：(1) 用户侧的产品叙事要和 Meta 式的"你给我们训练数据"**明确切割**——mem-ai 的 memory 必须归用户所有、可导出、可删除、不训练任何共享模型；(2) 本地优先 / 端侧存储不是极客情怀，是下一代用户的基础预期——Meta 事件会持续给"你的数据去了哪"加温，独立创业者可以把它作为默认前提来设计产品。

## ProductHunt 2026-04-21 榜单

### RankAI｜445v ｜#1

自称"世界第一个 AI-native Agency for organic growth"。核心价值主张：**不是 SEO 工具，而是 AI 自动化的 SEO/GEO 代理公司（Agency-as-a-Software）**——客户不是买仪表盘自己干，而是把"获客"外包给 AI agent 流水线。YC 孵化项目，同时覆盖传统 SEO（Google）和 GEO（让 ChatGPT/Perplexity/AI Overviews 引用你）。

**怎么做：** 全链条 AI（关键词调研 → 内容生产 20+ 页/月 → 技术修复 → 持续 rewrite → 跟踪效果），定价 ~$500/月（传统 SEO Agency $3k–$15k/月）。卖"outcomes"不卖 dashboard。官方宣称已服务 100+ 企业。

**用户反馈：** PH 当天 launch，原生评论不够多。赛道外部信号——正面："AI search traffic converts at 4.4x the rate of traditional Google traffic"（bemodo.com）；负面：赛道拥挤（Profound $58.5M 融资、Bluefish、AthenaHQ、Adobe LLM Optimizer 都在抢这个位置），传统 SEO 圈担心"pure AI content at scale 会被 Google Helpful Content 惩罚"。**追踪至 2026-05-05 再补用户反馈。**

**【判断】对 mem-ai**：Personal OS / 跨 LLM 记忆层这种抽象叙事，传统 SEO 不好发力，**GEO 反而适配**——用户在 ChatGPT 里问"how to make AI remember me across apps"时被引用才是赢。产品上线前先建立 3–5 篇被 Perplexity/ChatGPT 引用的深度内容（技术博客 + HN + Reddit 交叉传播），比花 $500/月买 RankAI 更有效。Agency-as-a-Software 的商业模式本身值得注意：mem-ai 做增长时可以把"帮用户迁移/整理记忆到 LLM"做成这种模式。

### Twenty 2.0｜323v ｜#2

开源 CRM（AGPL-3.0，YC S23，44k+ GitHub stars）。2.0 从"开源 Salesforce 替代品"升级为 **"AI-native CRM SDK + 运行时"**——CRM 数据只是载体，真正卖点是让 AI agent 原生使用。

**怎么做：**
- **SDK-first**：`npx create-twenty-app` 脚手架。TypeScript 定义 custom objects、server-side logic、React 组件（渲染进 Twenty UI）、AI skills / agents、views、navigation
- **Native MCP Server**：每个 Cloud workspace 自带 MCP server，AI assistant（Claude/ChatGPT）通过 OAuth 连接，自然语言 read/write CRM
- 技术栈：React + NestJS + GraphQL；开源 + 自部署免费，Cloud 版收费
- 社区规模：44k stars，300+ active contributors，11,590+ commits

**用户反馈：**
- 正面："clean, intuitive interface and the rapid development pace"；"10–50 用户团队 + 有开发者做 setup 的话 production-ready"（sentisight.ai）
- 负面："paying customers reported disappointing experiences, customer support takes days to respond"；"Salesforce AppExchange has thousands of integrations, Twenty doesn't"

**【判断】对 mem-ai（重要）**：传统 SaaS 基座是 UI + REST API，AI 原生 SaaS 基座是 **MCP Server + TypeScript SDK + AI skills**。Twenty 2.0 不是给人用的 CRM，是给 AI agent 用的 CRM——人类是审核者和编排者。**mem-ai 的"跨 LLM 记忆层"如果只做 Chrome 插件或原生 app，就是给人用的工具；要真正成为 Personal OS 底层，必须提供 MCP Server 让任何 LLM / agent 通过标准协议读写用户记忆**。这才是"越用越懂他"在 agent 时代的实现方式。`npx create-twenty-app` 脚手架 + React 组件渲染 inside 主应用这套模式，mem-ai 可以直接抄。

### Kimi K2.6｜301v ｜#3

Moonshot AI 开源大模型新版本。**1T 总参 / 32B 激活 MoE**，多模态，Modified MIT License。核心定位：**长时程自主编码 + Agent swarm 编排的开源 SOTA**，直接对标 Claude Opus 4.6/4.7 和 GPT-5.4。

**怎么做：**
- **Benchmark**：HLE-Full with tools 54.0（第一，超 GPT-5.4 的 52.1、Opus 4.6 的 53.0、Gemini 3.1 Pro 的 51.4）；SWE-Bench Pro 58.6（Opus 4.6 max 53.4）；SWE-Bench Verified 80.2；LiveCodeBench v6 89.6
- **Agent Swarm 架构**：300 sub-agents 并行，4,000 coordinated steps（K2.5 是 100 agents / 1,500 steps）
- **长时程演示**：13 小时自主执行，迭代 12 个优化策略，1,000+ tool calls，修改 4,000+ 行代码。Reddit 实测：5 天自主 infra agent run，kernel 重写，Zig 推理引擎比 LM Studio 快 20% TPS
- **定价杀手**：$0.95/M input，$4.00/M output，$0.16/M cache-hit input。**比 Claude Sonnet 4.6（$3/$15）便宜 5–10 倍**

**用户反馈：**
- 正面："viable Claude/GPT backend for coding and infra work"；Reddit 早期测试者称其 "Opus-flavored" reasoning；36kr 称"在博士级 HLE 以 54.0% 居首；指挥 300 个 Agent 实测手搓 3D 格斗游戏"
- 负面：latent.space 质疑"The lead over GPT-5.4 is 0.9 percentage points - within noise of evaluation variance"；VentureBeat："runs agents for days — and exposes the limits of enterprise orchestration"——**模型能力已超过 orchestration 工具的成熟度**；钛媒体：K2.6 API 定价比 K2.5 涨了 58%，质疑"低价开源"叙事能否持续

**【判断】对 mem-ai**：
1. **Opus 4.7 replacement 叙事成立，但要区分"模型替代"和"agent 替代"**：纯模型场景可以替换，Claude Code 的 hooks/MCP/skills 生态还有工具链护城河。建议：**新项目用 Kimi 起步，存量项目按月度 API 账单超过 $500 再迁移**
2. **Agent swarms 是真工具不是噱头**：300 sub-agents / 13 小时不是 demo。但 orchestration 成熟度不够——**下一个热点可能是 agent swarm 的编排、监控、成本控制工具**（类似 2024 的 LangSmith / Helicone）。mem-ai 记忆层可以扩展成"agent swarm 的长期记忆层"
3. **用户行为范式迁移**：用户以后不是自己打字，而是**监督 agent 替他做事**。"让 LLM 越用越懂他"要扩展到"让 agent 越用越懂他"——mem-ai 的记忆模型不能只存对话和偏好，还要存**用户对 agent 行为的反馈**（approval/rejection/edit），这是下一代 Personal OS 的关键差异化

### Chronicle by OpenAI ｜133v ｜#9 ｜**直接威胁项**

> Build Codex memories from recent screen context

**是什么：** OpenAI 给 Codex（macOS 应用）新增的 screen-context memory 功能。本地周期性截屏 → 云端 Codex session 处理（OCR + 时间信息 + 文件路径）→ 生成 memory summaries → 存回本地。下次 prompt Codex 自动读取，减少重述 context。Sam Altman 内部代号 **"Telepathy"**。

**怎么做：** Opt-in research preview，**仅 ChatGPT Pro（$200/月）在 macOS 上可用**，EU/UK/CH 暂不开放。需要 Screen Recording + Accessibility 系统权限。数据流：截图本地 → OpenAI 服务器做 OCR + summarization → 返回文本 summary → 存本地（unencrypted）。

**对 mem-ai 的威胁判断：**

**结构性相撞：Yes。** Chronicle 做的事 = "screen recording → semantic memory for coding assistant"，和 mem-ai 的"让 LLM 越用越懂他"在叙事上同源。Sam Altman 说"Telepathy"——这就是用户想让 AI 达到的终态。

**定位差异明显：**
- Chronicle 绑定 OpenAI Codex（单一 LLM、编程场景、macOS 付费用户）
- mem-ai 是 **cross-LLM + 跨场景（不只编程）+ 普通用户（不只 developer）**
- Chronicle 的隐私设计是重大负债：cloud processing + unencrypted local memory + EU 不可用 + prompt injection 风险。Windows Recall 就是前车之鉴
- 替代叙事："我们不需要一个 OpenAI 的 telepathy，我们需要一个**用户自主控制的 telepathy**"

**真正的威胁不是功能撞车，是用户预期被定型。** Sam Altman 站台 + OpenAI 渠道覆盖 + ChatGPT Pro 用户规模——**这会让"AI 自动理解我"的主流认知锁定在"OpenAI 看我屏幕"这个版本**。

**三条行动启示：**
1. **加速推出一个能用的 MVP**：Chronicle 已经把"screen → memory → LLM context"做出来了，mem-ai 同类能力要有可演示版本
2. **在定位上明确反 Chronicle**：本地优先、加密、跨 LLM、开源。**正面打 OpenAI 渠道优势是送死**
3. **把"screen recording"作为可选高级模式**，不作为主入口。主入口应该是对话历史导入 / Chrome 插件捕获用户和 ChatGPT/Claude 的对话 / 用户主动标注——**让用户感到自己是主人，不是被看的**
