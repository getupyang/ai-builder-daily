今日最大的事：法国政府全面弃用 Windows 转向 Linux

HN 502分 / 634评论（另有一条同主题 835分）。

法国数字事务部（DINUM）于4月8日正式宣布，将政府工作站从 Windows 迁移至 Linux，并要求所有政府部委在2026年秋季前提交"去美国化"实施方案，涵盖操作系统、协作工具、安全软件、AI、数据库、云基础设施等八大类别。影响范围最终覆盖250万公务员。具体采用哪个 Linux 发行版尚未公布。

为什么这么热：

同时命中了 HN 社区的多个敏感神经——Linux vs Windows 经典之争、地缘政治下的数字主权、政府大规模技术迁移的可行性辩论。634条评论说明社区在"能不能做到"这个问题上严重分裂。

各方声音： - 法国部长 David Amiel： "我们必须重新掌控数字命运。我们不能接受数据和战略决策依赖于我们无法控制规则和定价的解决方案。" - 乐观派： "Linux 已完全成熟。如果法国做成了，将为其他政府打开18-24个月的跟进窗口。" - 怀疑派： "他们说要换成'Linux操作系统'这个表述本身就是死亡 flag。真正的难题不是技术，是让管理层和用户接受改变。" - 微软： TechCrunch 求证时拒绝评论。

背景： 法国宪兵队（Gendarmerie）从2004年起逐步迁移，自研 GendBuntu（基于 Ubuntu），目前97%的工作站（超10.3万台）运行 Linux，每年节省约200万欧元许可费，TCO 降低40%。2026年1月法国已要求250万公务员从 Teams/Zoom 切换到自研 Visio 平台（基于 Jitsi）。DINUM 明确将宪兵队案例作为全国推广的治理模板。

【判断】这不是又一次"政府说要用 Linux"的空头支票。三个关键不同：一是宪兵队10万台机器、20年成功先例作为内部背书；二是特朗普时代地缘紧张提供了真实政治动力；三是有强制时间线（秋季提交方案）而非自愿参与。真正的挑战在执行——250万公务员的培训、专业软件兼容性、IT 支持体系重建。宪兵队花了20年，全政府规模不会更快。但方向已不可逆。

【判断】这是 AI 行业反对情绪从网络走向现实暴力的标志性事件。Altman 的回应将事件引向 AI 治理讨论，但也被部分人视为公关操作。不管怎样，这条时间线值得所有 AI 从业者警醒。

可在线对弈的国际象棋变体，棋盘只有一条线，每方三颗棋子（国王、骑士、车），有 AI 对手。口号是"去掉那些多余的维度"。HN 社区对"把复杂事物降维到本质"这类创意非常买账。

macOS 隐私设置不可信（460p / 158c）

macOS 的"隐私与安全"面板存在误导性 bug——当应用通过"打开/保存"对话框获得受保护文件夹的访问权后，沙箱限制实际上已被解除，但系统界面仍显示"已阻止"。你看到的"阻止"可能根本没生效。

WireGuard 新版发布，微软签名问题已解决（472p / 139c）

与上周 VeraCrypt 封号事件呼应。WireGuard 向微软提交新驱动签名时账户被临时封禁，曝光后一天内解封。WireGuard 作者称这只是"官僚流程失控"。但模式已经清晰：微软的驱动签名流程对开源项目存在系统性风险。

Linux 内核对 AI 辅助编码的官方政策（317p / 213c）

Linux 内核正式文档明确：欢迎 AI 辅助贡献，但只有人类能签署 DCO；开发者必须完整审查 AI 生成代码并承担全部责任；使用 AI 工具需添加 "Assisted-by" 标签。这是全球最重要的开源项目对 AI 编码的正式立场——不禁止，但责任完全归人。

ProductHunt 2026-04-10 榜单

Claude Advisor Tool｜373v｜Opus 当顾问，Sonnet/Haiku 当执行

是什么： Anthropic 推出的多模型协作功能，让 Opus 作为"顾问"指导 Sonnet 或 Haiku 执行任务，实现高质量推理与低成本执行的分层配合。
解决什么问题： 单独用顶级模型成本高、速度慢；单独用小模型质量不够。
怎么做： Opus 负责规划、审查、纠错，Sonnet/Haiku 负责执行编码或写作；用户在 Claude Code CLI 中配置切换；本质是 orchestrator-worker 架构。
用户反馈： 评论区开发者普遍反馈"代码质量明显提升，token 成本降低 50%+"；部分用户提到 advisor 延迟偶尔较高。
【判断】这是 Anthropic 对"模型即平台"的关键一步——不只卖单个模型，而是卖模型组合的工作流。对独立开发者直接利好：用 Opus 把关架构、Haiku 干活，性价比最优。

Integrations in Spine｜303v｜跨应用 AI 信息整合

是什么： AI 驱动的跨应用信息整合工具，连接 Notion、Slack、Gmail、Linear 等，自动综合和研究分散在各处的信息。
解决什么问题： 知识工作者信息分散在十几个工具里，AI 助手只能访问单一数据源。
怎么做： 一键授权连接多应用，构建统一知识索引；自然语言提问，跨应用检索并综合答案；支持自动监控变更。
用户反馈： 早期用户称"终于不用在 Slack 和 Notion 之间来回跳了"；也有人担心数据隐私和权限粒度。
【判断】跨应用整合是刚需但也是竞争红海（Glean、Dust、Mem）。差异化要看整合深度。对一人公司：如果能替代手动在多工具间搬运信息，值得试用。

SoulLink｜297v｜移动端 3D Vibe Coding

是什么： 移动端 3D Vibe Coding 伙伴，通过自然语言在手机上生成和编辑 3D 场景、动画和简单游戏原型。
解决什么问题： 3D 开发门槛极高（Unity/Blender 学习曲线陡峭），移动端几乎没有 3D 创作工具。
怎么做： 语音/文字描述生成 3D 场景；内置代码生成引擎+实时预览+手势交互。
用户反馈： 新发布暂无深度反馈，PH 评论以"很酷"、"概念新颖"为主。建议追踪。
【判断】"移动端 + 3D + Vibe Coding"三概念叠加，噱头强但实用性存疑。核心问题：谁会在手机上做 3D 开发？如果定位为创意草稿工具（3D 版 Napkin）可能有场景。
